数据库面试题

2020-03-10 8 min read # MySQL

DQL，DML，DDL，DCL

数据查询语言DQL

数据查询语言的基本结构是select ... from ... where ...

数据操纵语言DML

主要有三种形式：insert;delete;update

数据定义语言DDL

用来创建数据库中的各种对象：表，视图，索引，同义词等：create table/view/index/syn

DDL是隐形提交的，不能回滚。

数据控制语言DCL

授予或者回收访问数据库的某种特权：grant;rollback;commit

什么是存储过程？

存储过程就像是编程语言中的函数一样。封装了我们的代码。

存储过程的优点
  1. 能够将代码封装起来
  2. 能保存在数据库中
  3. 让给编程语言进行调用
  4. 存储过程是一个预编译的代码块，执行效率比较高
存储过程的缺点
  1. 每个数据库的存储过程语法几乎都不一样，十分难维护
  2. 业务逻辑放在数据库上，难以迭代

表设计的三个范式是什么

  1. 第一范式：字段是最小的单元不可再分。

    • 学生信息组成学生信息表，有年龄，性别，学号等信息组成。这些字段不可再分。

  2. 第二范式：满足第一范式，表中的字段必须完全依赖于全部主键而非部分主键。

    • 其他字段组成的这行记录和主键表示的是同一个东西，而主键是唯一的，他们只需要依赖于主键。
    • 学号为110的同学，姓名为zhao，年龄是28。姓名和年龄都依赖着学号主键

  3. 第三范式：满足第二范式，每一列数据都和主键直接相关，而不是间接相关

    • 就是数据只在一个地方存储，不重复出现在多张表中，可以认为就是消除传递依赖
    • 比如，在大学里分了很多系。这个系别管理表信息有以下字段组成：系编号，系主任，系简介，系架构。那么能不能在学生信息表中添加系编号，系主任等信息呢？不行！因为这样就冗余了，非主键外的字段形成了依赖关系（依赖到学生信息表了）。正确的做法是：学生表只能增加一个系编号字段。

什么是视图，以及视图的使用场景

视图是基于数据表的一种虚拟表。

  1. 视图是建立在已有表的基础上。视图赖以建立的这些表为基表
  2. 向视图提供数据内容的为select语句，可以将视图理解为存储起来的select语句
  3. 视图没有存储真正的数据，真正的数据还在基表中
  4. 虽然操作的是视图，但最终视图还会专程操作基表
  5. 一个基表可以有0个或多个视图

视图用起来安全，方便，但不提高查询效率

  • 用户想看什么数据，就提供什么数据。保证其他的数据不会泄露
  • 有时候SQL语句会特别长，用视图封装起来再使用就会很方便
  • 视图可以让我们专注逻辑，但不能提高效率

drop,delete,truncate分别在什么场景下使用

drop：属于DDL；不可回滚；不可带where；表内容和结构删除；删除速度快

truncate：属于DDL；不可回滚；不可带where；表内容删除；删除速度快

delete：属于SML；可回滚；可带where；表结构在，表内容要看where执行情况；删除速度慢，逐行删除

  1. 不需要一张表的时候用drop
  2. 保留表结构但删除所有数据的时候用truncate
  3. 想删除部分数据的时候用delete from ... where ...

索引是什么？

什么是索引？
  1. 一种快速查询表中内容的机制，类似于新华字典的目录
  2. 运用表中某些字段上，但存储时，独立于表外
索引将数据编程有序的
快速定位到硬盘中的数据文件

什么是事务？

简单的说：一个session中所进行所有的操作，要么同时成功，要么同时失败

ACID四个基本要素
  1. 原子性：整个事务要么全部完成，要么全部步完成。如果中间执行错误，会被rollback到事务开始前的状态。
  2. 一致性：如果事务是并发多个，系统也必须如同串行事务一样操作，其主要特征就是保护性和不变性。以转账为例，假如现在A和B共有100元，事务执行完毕后，A和B的账户里页应该是100元，不能多也不能少。
  3. 隔离性：隔离状态执行事务，是他们好像是系统在给定时间内执行的唯一操作。如果有两个事务，运行在相同的时间内，执行相同的功能，事务的隔离性将确保每一事务在系统中认为只有该事务在使用系统。这种属性有时候被称为串行化。为了防止事务操作时间的混淆，必须串行化或者序列化的请求。使得同一时间有且仅有一个请求用于同一数据。
  4. 持久性：事务完成后，该事务对数据库所做的更改便持久的保存在数据库中，不会被回滚。

SQL约束有哪几种？

  1. not null：用于控制字段的内容一定不为空
  2. unique：控制字段内容不能重复，一个表允许多个unique约束
  3. primary key：控件字段内容不能重复，但在一个表中只能出现一个
  4. foreign key：用于预防破坏表之间链接的动作，也能防止非法数据插入外键，因为它必须是它指向的那个表中的值之一
  5. check：用于控制字段的范围

varchar和char的区别

char是一种固定长度的类型；varchar长度可变

mysql中有关权限的表有哪几个？

  1. user权限表：记录允许连接到服务器的用户账户信息，里面的权限是全局的
  2. db权限表：记录哥哥账号在各个数据库上的操作权限
  3. table_priv权限表“记录数据表级的操作权限
  4. columns_priv权限表：记录数据列级的操作权限
  5. host权限表：配合db权限表的时候对给定主机上数据库级别操作权限做更细致的控制。这个权限不受grant和revoke的影响

数据库的优化思路

选择最有效率的表名顺序

数据库的解析器按照从右到左的顺序处理from中的表。写在最后的将被最优先处理

在from中包含多个表的情况下：

  • 如果表是完全无关的情况下，将记录和列名最少的表写在最后，然后依次类推
  • 如果表有关系的话，将引用最多的表，放在最后，以此类推
where子句连接顺序

数据库采用自右向左的解析顺序，根据这个原理，表之间的连接必须卸载where条件的左边，那些可以过滤掉最大数量记录的条件必须写在右边

SELECT
	DISTINCT s1.s_name 
FROM
	student AS s1,
	score AS s2 
WHERE
	s1.s_id = s2.s_id 
	AND s2.score > 50;

因为s2.score>50可以过滤掉大量数据，所以放在右边

select语句中避免使用*
  • 消耗时间多
  • 不直观
用truncate代替delete

delete是一条条删，truncate是除了表结构删除所有内容

多使用内部函数

比如concat()就会比||快，因为内部函数已经被优化过

多使用commit

commit会释放回滚点

善用索引

索引就是为了提高查询效率

SQL大写

在编写SQL语句的时候，要用大写来写关键词，因为服务器总是先把小写转换成大写后才执行

避免在索引列上使用not

服务器遇到not后，会停止目前的工作，转而执行全表扫描

避免在索引列使用计算

where子句中如果索引列是函数的一部分，优化器将不使用索引而使用全局扫描，这样会变得很慢

  • 用>=代替>
-- 低效：首先定位到score=100的记录并扫描到第一个大于100的记录
SELECT s_id FROM score WHERE score>100
-- 高效：直接定位到第一个score=100的记录
SELECT s_id FROM score WHERE score>=101
  • 用in代替or
SELECT * FROM score WHERE s_id='01' OR s_id="02";

SELECT * FROM score WHERE s_id IN ("01","02");
