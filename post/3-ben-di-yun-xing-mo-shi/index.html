3 - 本地运行模式

2020-03-14 3 min read

配置Hadoop-env.sh文件

# 进入hadoop文件夹
cd /home/parallels/Desktop/module/hadoop-2.9.2/etc/hadoop

# 打开hadoop-env.sh文件
vim hadoop-env.sh
测试是否成功
# 在 /home/parallels/Desktop/module/hadoop-2.9.2目录下执行
bin/hadoop

测试本地模式

mkdir input
cp etc/hadoop/*.xml input
bin/hadoop jar hadoop/mapreduce
# 输出的文件夹必须是一个不存在的文件夹
bin/hadoop jar share/hadoop/mapreduce/hadoop-mapreduce-examples-2.9.2.jar grep input output 'dfs[a-z.]+'

# 查看结果
cat output/*

如果最后结果如图所示，则表明hadoop的环境已经打通。类似于「hello」

官方WordCOunt例子

# 新建一个文件
mkdir input
cd input
vim wc.input

开始执行创建好的wcinput文件

#执行文件
bin/hadoop jar share/hadoop/mapreduce/hadoop-mapreduce-examples-2.9.2.jar wordcount wcinput wcoutput

#查看文件结果
cat wcoutput/*

以上就是本地执行模式。

配置集群

Hadoop-env.sh

已配置

Core-site.xml

#进入core-site.xml文件
vim core-site.xml

将以下内容加进去

<!-- 指定HDFS中NameNode的地址-->
<property>
    <name>fs.defaultFS</name>
    <value>hdfs://slave2:9000</value>
  </property>

<!--指定Hadoop运行时产生文件的存储目录-->
  <property>
    <name>hadoop.tmp.dir</name>
    <value>/home/parallels/Desktop/module/hadoop-2.9.2/data/tmp</value>
  </property>

Hdfs-site.xml

#进入hdfs-site.xml文件
vim hdfs-site.xml

将以下内容加进去

<!-- 指定HDFS副本数量，单一节点至多存储一份副本 -->
<property>
            <name>dfs.replication</name>
            <value>1</value>
 </property>
格式化namenode
bin/hdfs namenode -format
启动namenode
hadoop-daemon.sh start namenode

# 验证
jps
